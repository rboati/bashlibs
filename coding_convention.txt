
= Bash coding convention

== Variables
- Always declare explicitely a variable to flag the point in which it's created.
- Use "declare -g" explicitely for global variables, even in the main scope.
- Use "local" for local variables.
- Use UPPERCASE identifiers only for global variables.
- Use lowercase identifiers for local variables.

== Functions
- Use the short form for definning a function:
	function_name + '()' + {
- Use lowercase identifiers for function names.
	Exception for the prefix/namespace part.
- User only alfanumeric or '_' chars, avoid punt chars ( [.-@:] )
	Exception is used as separators for the namespace part.

== Statements
- Try to avoid long compound statements.
